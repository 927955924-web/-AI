# 智语AI客服系统 - 软件架构与资源说明文档

本文档详细描述了智语AI客服系统的技术架构、目录结构及UI资源管理规范，旨在帮助开发人员理解系统设计并指导后续的图标变更与UI定制。

## 1. 系统架构概览

本系统采用分层架构设计，主要包含以下层次：

*   **UI层 (Presentation Layer)**: 基于 `tkinter` 和 `ttk` 构建的桌面图形界面。负责用户交互、界面展示及事件响应。
*   **服务层 (Service Layer)**: 包含核心业务逻辑，如 AI 对话服务 (`AIService`)、浏览器自动化管理 (`BrowserManager`) 等。
*   **领域层 (Domain Layer)**: 定义核心业务实体（如 User, Shop, Product 等）。
*   **基础设施层 (Infrastructure Layer)**: 处理数据持久化（SQLite）、日志记录、文件路径管理及安全存储。

## 3. UI资源与图标变更指南

为了方便后续修改软件图标及界面图片，建议遵循以下规范：

### 3.1 图标资源存放位置
建议在项目根目录下创建 `assets/icons` 目录，统一存放所有图标文件。

*   **主程序图标**: `assets/icons/app_icon.ico` (用于EXE文件图标及窗口左上角图标)
*   **功能按钮图标**: 建议使用 `assets/icons/toolbar/` 等子目录分类存放。

### 3.2 代码中引用图标
在 `src/ui/app.py` 中，可以通过以下方式加载和使用图标：

```python
import os
from tkinter import PhotoImage

# 定义资源路径获取函数
def get_resource_path(relative_path):
    if hasattr(sys, '_MEIPASS'):
        return os.path.join(sys._MEIPASS, relative_path)
    return os.path.join(os.path.abspath("."), relative_path)

# 示例：设置窗口图标
icon_path = get_resource_path("assets/icons/app_icon.png")
img = PhotoImage(file=icon_path)
self.root.iconphoto(False, img)
```

### 3.3 如何变更图标
1.  **准备新图标**: 制作新的 `.ico` 或 `.png` 文件。
2.  **替换文件**: 将新文件覆盖 `assets/icons/` 下的同名文件。
3.  **重新打包**: 如果涉及EXE图标变更，需要重新运行 PyInstaller 打包命令。

### 3.4 修改打包配置 (.spec)
如果在代码中使用了新的静态资源目录（如 `assets`），需要在 `电商客服.spec` 文件中添加数据文件映射：

```python
datas=[
    ('assets', 'assets'),  # 将 assets 目录打包进 EXE
    # ... 其他数据文件
],
```

## 4. 关键模块说明

*   **src/ui/app.py**: 系统的核心UI文件。包含主窗口布局 (`DesktopApp` 类)、登录对话框 (`ensure_login` 方法) 以及各功能模块的初始化逻辑。所有的界面修改（如颜色、布局、按钮样式）均在此文件中进行。
*   **src/infrastructure/db.py**: 数据库初始化脚本。定义了 SQLite 表结构。如果需要新增字段，需修改此处的 `CREATE TABLE` 语句。
